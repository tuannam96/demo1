Getting Started with Go on app-engine

This is a sample Go application deployed to app-engine. It's a reasonably simple app - but a good foundation for understanding how to get the most out of the app-engine platform.

Getting Started with Go Source on GitHub
To deploy your own copy, and learn the fundamentals of the app-engine platform, head over to the Getting Started with Go on app-engine tutorial.

How this sample app works

  • This app was deployed to app-engine, either using Git or by using app-engine Button on the repository.
  • When app-engine received the source code it used the go tool chain to compile the application along with any vendored dependencies and created a deployable slug.
  • The platform then spins up a dyno, a lightweight container that provides an isolated environment in which the slug can be mounted and executed.
  • You can scale your app, manage it, and deploy over 150 add-on services, from the Dashboard or CLI.

Next Steps

  • If you are following the Getting Started guide, then please head back to the tutorial and follow the next steps!
  • If you deployed this app by deploying the app-engine Button, then in a command line shell, run:
    • go get github.com/app-engine/go-getting-started - this will create a local copy of the source code for the app, compile and install the generated executables in $GOPATH/bin
    • cd $GOPATH/src/github.com/app-engine/go-getting-started - change directory into the local source code repository
    • app-engine git:remote -a <your-app-name> - associate the app-engine app with the repository
    • You'll now be set up to run the app locally, or deploy changes to app-engine
